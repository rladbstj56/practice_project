# ☕️ 스타벅스 고객 데이터 분석 파이프라인

---

## 🔬 분석 방법론 트리

```
분석 파이프라인
│
├── 📦 전처리 (preprocessed_fin.ipynb)
│   ├── 결측치 처리
│   │   ├── age 이상치(118) → NaN
│   │   ├── gender 결측 → 'Unknown'
│   │   └── income 결측 → 0 + is_profile_missing 플래그
│   ├── 파생변수 생성
│   │   ├── 채널 one-hot encoding (ch_web / ch_email / ch_mobile / ch_social)
│   │   ├── reward_ratio = reward / difficulty
│   │   ├── offer_strength = reward - difficulty
│   │   ├── age_group (10세 단위 범주화)
│   │   ├── age_gender (연령대 × 성별 조합)
│   │   └── income_group (4구간 범주화)
│   └── 병합: transcript ← portfolio ← profile (left join)
│
├── 🔍 EDA (eda.ipynb)
│   ├── 파생변수 추가
│   │   └── join_year / join_month / join_cohort (가입일 분해)
│   ├── 퍼널 수치 확인
│   │   └── 수신 → 열람 → 완료 단계별 고객 수
│   └── 미인지 완료 탐색
│       └── viewed 없이 completed → is_aware 플래그
│
├── ⏱️ Act 1 — 시간 흐름 (stage1.ipynb)
│   ├── 인스턴스 생성
│   │   └── merge_asof (시간 기반 비동등 조인)
│   │       received → viewed → completed 순서 매핑
│   ├── 기술통계
│   │   ├── 수신→열람 / 수신→완료 반응 시간 (평균 · 중앙값 · 분위수)
│   │   └── 유효기간 내 완료율
│   └── 자연 거래 분리
│       └── 오퍼 수신 기간 외 거래 → natural transaction 정의
│
├── 📊 Act 2 — 퍼널 통계 (stage2.ipynb)
│   ├── 정규성 검정
│   │   └── Shapiro-Wilk (최대 5,000 샘플)
│   ├── 비모수 검정
│   │   └── Mann-Whitney U test (bogo vs discount 반응 시간)
│   ├── 효과 크기
│   │   └── rank-biserial r
│   └── 세그먼트별 미열람 / 미인지 완료율 집계
│
└── 🧩 Act 3 — 세그먼트 (stage3.ipynb)
    ├── 독립성 검정
    │   └── 카이제곱 검정 (χ²) + Cramér's V (효과 크기)
    ├── 집단 간 평균 비교
    │   └── 일원배치 ANOVA
    ├── 사후 검정
    │   └── Tukey HSD (모든 구간 쌍 비교)
    └── 히트맵 분석
        └── 세그먼트 × 오퍼 유형 완료율 교차표
```

## 💡 분석 흐름별 인사이트

### EDA — 고객 기반 파악

> "우리가 생각한 스타벅스 고객과 실제 데이터가 다르다"

- 주 이용층은 2030이 아닌 **5060** (60대 이상 34.6%로 최다)
- 소득은 **5-7.5만 구간**이 37.9%로 집중, 평균 $65,405
- 거래 중앙값 $8.89 → 개인 음료 소비 패턴 (대량 주문 영향 미미)

### Stage 1 — 시간 흐름

> "오퍼 설계는 잘 됐다. 문제는 '누가 반응하느냐'다"

- 수신 후 **24시간 내** 66%가 첫 열람 → 초기 반응이 핵심
- 완료는 **7일 내** 96.7% → 유효기간 설계 적절
- 자연 거래 vs 오퍼 중 거래 금액 차이 $0.02 → **오퍼가 소비 금액을 올리지는 않음**
- 오퍼 없이도 완료하는 고객 **17%** → reward 비용 낭비 구간

### Stage 2 — 퍼널 병목

> "열람이 안 되는 것이 문제, 완료 안 되는 건 세그먼트 문제"

- 수신→열람 **75.7%** / 열람→완료 **57.3%** → 열람 단계가 첫 번째 병목
- 미열람 24.3% 중 **저소득·discount·informational** 과대 대표 → 채널/메시지 재설계 필요
- 미인지 완료 고객 거래금액이 오히려 더 높음 ($20.37 vs $18.05) → **충성 고객, 오퍼 불필요**
- bogo가 discount보다 **0.8일 빠르게** 완료 (Mann-Whitney, 중간 효과)

### Stage 3 — 세그먼트

> "누구에게 어떤 오퍼를 줄지가 명확해진다"

- **소득이 가장 강한 변수** (V=0.379) — 고소득일수록 완료율·거래금액 모두 압도적
- **여성이 모든 오퍼에서 남성보다 14~19%p 높은 완료율** → 남성 타겟 별도 전략 필요
- **나이 많을수록 완료율 높음** — 5060이 오퍼에 가장 잘 반응
- **신규 가입자(2018) 완료율 34.2%** → 온보딩 전용 오퍼 필요

## 🎯 최종 전략

> 오퍼를 잘 받는 사람 **(고소득·여성·5060·장기회원)** 에게 집중하고,
> 오퍼 없이도 사는 충성 고객에겐 **reward 낭비를 줄여라.**
> 신규·저소득·남성에겐 **다른 설계가 필요하다.**
